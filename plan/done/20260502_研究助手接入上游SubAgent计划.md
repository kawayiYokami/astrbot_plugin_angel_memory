# 研究助手接入上游 SubAgent 计划

## 背景

上游 AstrBot 已新增实验性的 SubAgent / Handoff 系统。当前插件的研究助手由 `tools/research_tool.py` 自己实现 `research_topic` 工具，并在工具内部手动调用 `context.tool_loop_agent()`。

本计划目标不是兼容并存，而是将研究助手切换为上游 Handoff 子代理入口，移除旧的 `research_topic` 暴露面。

## 目标

1. 取消向主 LLM 注册 `research_topic`。
2. 注册一个上游 Handoff 子代理工具 `transfer_to_researcher`。
3. 复用现有研究员提示词 `tools/research_fellow_prompt.md`。
4. 复用现有 `research_assistant` 配置中的模型与人格设定。
5. 保持失败可降级：如果当前 AstrBot 版本没有 `Agent` / `HandoffTool`，插件不应整体初始化失败。

## 非目标

1. 不修改 AstrBot 全局 `subagent_orchestrator` 配置。
2. 不要求用户去 WebUI 手动创建 SubAgent。
3. 不保留旧工具名 `research_topic`。
4. 不在第一版实现每次调用动态 `complexity -> max_steps`，因为上游 Handoff 当前使用 provider 全局 `max_agent_step`。

## 注册方式

在插件启动注册 LLM 工具时：

1. 加载 `tools/research_fellow_prompt.md`。
2. 读取插件配置 `research_assistant`。
3. 创建上游 `Agent(name="researcher", instructions=..., tools=...)`。
4. 包装为 `HandoffTool(agent=..., tool_description=...)`。
5. 设置 `handoff.provider_id`：
   - `research_assistant.provider_id` 非空：固定使用该模型提供商。
   - 为空：不设置，由上游执行链路使用当前会话模型。
6. 通过 `context.add_llm_tools(...)` 与其他记忆工具一起注册。

主 LLM 最终看到：

```text
core_memory_remember
core_memory_recall
note_recall
transfer_to_researcher
```

不再看到：

```text
research_topic
```

## 模型策略

研究子代理使用的模型由 `provider_id` 决定：

1. 优先使用插件配置 `research_assistant.provider_id`。
2. 未配置时跟随当前会话 provider。

这样与当前 `research_topic` 的模型选择逻辑保持一致。

## 工具集策略

优先沿用当前研究人格配置：

1. `research_assistant.persona_id` 非空且人格存在：
   - 将人格 prompt 追加到研究员系统提示词。
   - 使用该人格配置的工具列表。
   - 如果人格工具为 `None`，交给上游 Handoff 语义表示“所有工具”。
2. 未配置人格或人格不存在：
   - 子代理工具设为 `None`，表示使用所有已激活工具。

## 行为变化

1. 研究入口从普通工具调用改为上游 Handoff 委派。
2. 工具参数从 `topic` / `complexity` 变为上游默认的：
   - `input`
   - `image_urls`
   - `background_task`
3. 支持上游原生后台任务 `background_task=true`。
4. 不再发送旧 `start_messages` 随机启动提示，除非后续单独扩展。
5. 不再由插件工具包裹“请主 LLM 转述给用户”的提醒，改由 Handoff 返回结果后进入上游主代理流程。

## 兼容与降级

由于上游文档标注 SubAgent 仍是实验性功能，注册逻辑需要做导入保护：

1. 能导入 `astrbot.core.agent.agent.Agent` 与 `astrbot.core.agent.handoff.HandoffTool` 时，注册 `transfer_to_researcher`。
2. 无法导入时，跳过研究子代理注册，记录一条清晰 WARNING 日志。
3. 其他记忆工具仍正常注册。

## 修改范围

预计修改：

1. `main.py`
   - 移除 `ResearchTool` 的实例化和注册。
   - 增加研究 Handoff 子代理注册。
2. `tools/research_tool.py`
   - 删除或改造成轻量注册辅助模块。
3. `README.md`
   - 更新研究工具说明，从 `research_topic` 改为 `transfer_to_researcher`。
4. `AGENTS.md`
   - 更新工具列表与数据流描述。
5. `metadata.yaml`
   - 版本号递增。
6. `CHANGELOG.md`
   - 增加本次行为变更记录。

## 测试计划

1. 执行现有单元测试：

```powershell
python -m pytest -q
```

2. 静态检查注册结果：
   - 确认代码中不再注册 `research_topic`。
   - 确认 Handoff 子代理导入失败时不会影响其他工具注册。

3. 运行时验证：
   - 启动插件后日志显示已注册 `transfer_to_researcher`。
   - 主 LLM 工具列表不再包含 `research_topic`。
   - 调用 `transfer_to_researcher` 可完成研究任务。

## 风险

1. 上游 SubAgent / Handoff 仍是实验性功能，接口可能变动。
2. `complexity` 动态步数能力会消失，复杂研究深度依赖 provider 全局 `max_agent_step`。
3. `background_task` 行为由上游接管，最终唤醒和转述方式可能与旧工具不同。
4. 如果用户依赖旧工具名 `research_topic`，提示词或历史习惯需要同步更新。

## 验收标准

1. 插件启动成功。
2. 记忆工具仍正常注册。
3. `research_topic` 不再注册。
4. `transfer_to_researcher` 注册成功。
5. 配置了 `research_assistant.provider_id` 时，Handoff 子代理使用该 provider。
6. 未配置 provider 时，Handoff 子代理跟随当前会话 provider。
7. 用户生产环境测试反馈通过后，再归档本计划。
