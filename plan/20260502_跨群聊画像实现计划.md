# 跨群聊人物画像（事实画像）— 实现计划

> 日期: 2026-05-02
> 状态: 已实现/已验证

---

## 目标

在 angel_memory 中新增 user_profile scope，使 agent 在群聊中也能获知用户的称呼、身份、兴趣等长期事实画像。原始记忆层 scope 隔离不变，隐私边界在提取层。

## 新增文件

```
angel_memory/core/profile/
├── __init__.py
├── config.py          # 红绿灯标签配置（绿灯/黄灯/红灯）
├── extractor.py       # 画像提取器（规则粗筛 + LLM 精提 + 去重）
└── injector.py        # 画像注入器（recall 时合并 user_profile scope）
```

## 修改文件

- `tools/core_memory_recall.py`：recall 时额外查询 `raw/user_profile/{user_id}/` scope
- `main.py`：集成画像提取与注入

## 提取规则

绿灯（始终提取）：preferred_name, communication_style, public_identity, interest_domain, bot_feedback
黄灯（默认关）：personal_experience, emotional_state
红灯（零容忍）：密码/PII/亲密话题/"别跟别人说"

Pipeline：规则粗筛 → LLM 精提 → 同标签类型内去重(>0.85) → 更新覆盖

## 硬约束

- actor-aware：仅处理 sender_id == 目标用户 的消息
- 时态区分：文件名含 temporal 后缀（present/past）
- 提取异步不阻塞主流程
