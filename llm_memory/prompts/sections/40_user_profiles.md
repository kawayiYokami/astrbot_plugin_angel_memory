## 人物画像优先规则

画像是记忆系统中**最高优先级**的内容。只要对话中出现了能揭示群友稳定特征的信息，
就应优先生成画像记忆，再考虑其它类型。

### 画像为空时的硬底线

如果当前群友暂无已注入的画像记忆，这是最高优先级的记忆生成场景。
和群友聊了一整轮，明明能看出这个人是谁、关心什么、在做什么，
却因为「不确定是否值得记」而输出空的 memory_actions——这是不可接受的遗漏。

画像为空时，必须从本次对话中提炼至少一条用户画像。
前提：信息足够稳定、有明确对话依据、非敏感、非临时状态。
不允许为了补齐画像而编造或记录无依据内容。

### 自然推断即可记

不需要群友亲口说「我是 XX」。对话中自然流露的信息也算：
- 反复讨论某游戏深度机制 → 事实属性（深度玩家）
- 提到「我在杭州」→ 事实属性（所在地）
- 整个会话都在写 Rust → 技能树（Rust 开发者）
- 多次提到某个人的名字并表达亲近 → 关系图谱

### 1. 用户 ID 锚点
- 涉及具体群友时，必须以群友 ID（QQ号）作为 `tags` 锚点。
- `judgment` 中必须同时写出真实昵称和真实用户 ID，格式：`昵称（用户ID）`。
- 真实昵称和用户 ID 从对话历史中的 `[群友: 昵称（QQ号）]` 格式获取。
- 用户画像记忆的 `tags` 必须严格使用以下核心结构：
  - 群友真实昵称
  - 群友用户 ID
  - 画像属性
- 画像属性只能从以下 5 个标签中选择：
  - `用户别名`
  - `事实属性`
  - `技能树`
  - `关系图谱`
  - `活跃项目`
- 不要把大量细碎对象都塞进 `tags`；具体对象写进 `judgment` 和 `reasoning`。
- 不要只用昵称作为用户标识，因为昵称可能变化或重复。

### 2. 用户别名
- 记录群友稳定使用或明确指定的昵称、称呼、代号、常用名。
- 别名必须绑定用户 ID，避免昵称变化或重名导致画像混淆。
- 此类记忆的画像属性标签必须使用 `用户别名`，不要使用 `别名`。

### 3. 事实属性
- 记录群友主动透露、长期稳定、未来对话可复用的低敏事实属性。
- 事实属性应保持具体、可验证，不做性格推断。
- 包括但不限于：兴趣领域、所在地、职业、稳定偏好、语言习惯、长期要求。
- 禁止记录身份证、手机号、详细住址、密码、token 等敏感信息。

### 4. 技能树
- 记录群友已经掌握、正在学习或经常使用的技能、工具、语言、框架。
- 技能树应优先记录稳定能力，不记录一次性尝试。

### 5. 关系图谱
- 生成人物画像时，要优先识别稳定的关系边，格式上保持「群友 - 关系 - 对象」的清晰语义。
- 关系对象可以是具体人、群体、AI、人设、组织、项目、作品、地点或长期任务。
- 此类记忆的画像属性标签必须使用 `关系图谱`。
- 关系对象写入 `judgment`，不要额外扩展大量 `tags`。

### 6. 活跃项目
- 当群友正在持续推进某个项目、插件、作品、研究、长期任务时，应优先生成活跃项目画像。
- 活跃项目记忆必须回答：谁在推进、项目是什么、当前稳定目标或方向是什么。
- 此类记忆的画像属性标签必须使用 `活跃项目`。
- 项目名和当前方向写入 `judgment`，不要额外扩展大量 `tags`。
- 不记录一次性的临时待办，除非它属于长期项目的当前阶段。

### 禁止画像内容
- 禁止生成以下人物画像记忆：
  - 身份证号、护照号、社保号、银行卡号、手机号、详细住址等高敏身份或财务信息。
  - 密码、验证码、密钥、token、cookie、私密链接等认证信息。
  - 单次情绪、一次性口误、玩笑话、不可复用的临时状态。
  - 对群友的贬损性、武断性格判断、无依据推测。

### 画像冲突与修正
- 当新信息与旧用户画像冲突时，优先修正画像，不要长期并存互相矛盾的画像记忆。
- 发现同一群友在同一画像维度下有重复、过时或冲突记忆时：
  - 相似收敛优先使用 `merge`
  - 明确纠正旧事实优先使用 `updata`
- 若群友明确纠正旧画像，优先使用 `updata`，不能仅 `create` 一条新画像；且 `source_memory_ids` 只能写 1 条旧画像 ID。
- 合并或更新后的画像，`tags` 仍必须严格保持：真实昵称 + 用户 ID + 画像属性。

### 用户画像判定规则
- 只有同时满足以下条件的记忆，才视为用户画像记忆：
  - `tags` 中存在用户 ID
  - `tags` 中存在固定画像属性标签之一：`用户别名`、`事实属性`、`技能树`、`关系图谱`、`活跃项目`
- 群友昵称只用于可读性和辅助召回，不作为用户画像判定的唯一依据。
- 普通带人名的记忆不应被误判为用户画像。
- 普通带 `项目`、`关系` 等词的记忆不应被误判为用户画像。
