## 第二步：要不要记？五问筛选

按顺序过一遍，任何一问命中就不记：

1. **有明确对话依据吗？** → 找不到证据 → 不记
2. **是临时状态吗？** 临时情绪、单次口误、一次性操作步骤 → 不记
3. **是低价值噪音吗？** 寒暄、吐槽、按钮点击、发图发文件等元信息 → 不记
4. **是高敏信息吗？** 密码、身份证号、密钥、token → 不记
5. **是对群友的贬损性判断或性格推测吗？** → 不记

五问全过 → **准备记。**

> 核心原则：**宁可多记一条后续修正，也不要漏掉可能有用的信息。**
> 只要不是明显错误或高敏内容，有点不确定也先记下来。后续对话确认后可以合并或修正。

## 第三步：检查旧记忆

记之前先看已有记忆：
- 已有相似的 → **合并**（`merge`），让信息更准确、更完整、更简洁。
- 与旧记忆矛盾 → **修正**（`updata`）旧记忆，不新增矛盾条目。
- 群友明确纠正画像 → 只修正对应画像记忆，保持标签结构不变。
- 没有相关旧记忆 → 新增（`create`）。

## 记忆生成规则

### 强制要求（MUST）
1. 内容优先，不记过程。
2. 原子化，一条记忆只表达一个论断。
3. 只记录未来可能复用的信息。
4. 类型优先级：`knowledge > skill > event > emotional`。
5. 涉及具体群友时，`judgment` 必须同时写出真实昵称和用户 ID。
6. `tags` 要便于实体检索和知识图谱使用。
7. 发现重复时优先 `merge`，发现事实冲突修正时优先 `updata`。
8. 当对话中出现群友稳定偏好/属性/关系/技能/活跃项目时，优先生成画像记忆。
9. 引用原话或专有名词时用 `「」`，不要在 JSON 字符串内使用未转义的英文双引号。

### 禁止要求（MUST NOT）
1. 不记录礼貌语、按钮点击、发图发文件等低价值元信息。
2. 不记录 AI 的基础行为过程，如「AI 查询了……」「AI 总结了……」。
3. 不把问题本身直接记下来，应转成结论。
4. 不把多条论断塞进一条记忆。
5. 不输出 JSON 之外的内容。
6. 不记录高敏身份、财务、认证信息。
7. 不根据单次发言武断生成贬损性或推测性人物判断。
