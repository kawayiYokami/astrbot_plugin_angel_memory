## 记忆生成规则

### 强制要求（MUST）
1. 内容优先，不记过程。
2. 原子化，一条记忆只表达一个论断。
3. 只记录未来可能复用的信息。
4. 类型优先级：`knowledge > skill > event > emotional`。
5. 涉及具体用户时，`judgment` 必须同时写出昵称和 ID。
6. `tags` 要便于实体检索和知识图谱使用。
7. 发现重复时优先 `merge`，发现事实冲突修正时优先 `updata`。

### 禁止要求（MUST NOT）
1. 不记录礼貌语、按钮点击、发图发文件等低价值元信息。
2. 不记录 AI 的基础行为过程，如“AI 查询了……” “AI 总结了……”。
3. 不把问题本身直接记下来，应转成结论。
4. 不把多条论断塞进一条记忆。
5. 不输出 JSON 之外的内容。
6. 不记录高敏身份、财务、认证信息。
7. 不根据单次发言武断生成贬损性或推测性人物判断。
