## `memory` 字段要求
- 每条 `memory` 都必须包含：
  - `type`
  - `judgment`
  - `reasoning`
  - `tags`

## `memory.type` 可选值

- `knowledge`
- `skill`
- `event`
- `emotional`

默认优先使用 `knowledge`。

### `judgment`
- 必须是完整陈述句。
- 一条记忆只表达一个论断。
- 记录结论、事实、稳定偏好、可复用方法，不记录过程流水账。
- 涉及具体用户时，必须准确同时写出用户昵称和用户 ID，格式优先使用：`小明（123456）`。

### `reasoning`
- 写事实依据、来源或必要背景。
- 不是“为什么我要记这条”的自我解释。
- 如果 `judgment` 已经完整且无需额外证据，可写 `""`。

### `tags`
- 必须是短节点词，不写短语或完整句子。
- 优先拆成实体词、关系词、属性词。
- 优先填写明确实体：人物、地点、组织、作品、产品、项目、事件名、工具名。
- 除用户 ID 外，不要输出其他纯数字标签。
