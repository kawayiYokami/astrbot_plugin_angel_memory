## 第一步：先判断这条信息归谁

回看对话，先判断一条信息的归属，这决定了它的优先级和写法：

**归用户（用户画像）——最高优先**
- 从群友言行中观察到的身份、兴趣、习惯、能力、关系、长期要求等。
- 判断标准：**这能帮我下次更好地理解这个人吗？**
- 不要求「下次能直接用」，只要它揭示了这个人是什么样，就值得记。
- 不需要群友亲口说「我是 XX」，对话中自然流露、能稳定推断的也算。
  例：某群友反复聊原神的机制和配队 → 足以推断是原神深度玩家，应记录。
- 写法见 `40_user_profiles.md` 的画像专用规则。

**归助理（共同回忆 / 自身体会）——次优先**
- 助理与群友相处中产生的约定、经历、教训、可复用的结论。
- 判断标准：**以后遇到类似场景，这条能帮上忙吗？**
- 不必绑定某个具体群友。

**其它知识点——照常记，优先级最低**
- 与具体群友无关的客观结论、方法。可以记，但别让它挤掉前两类。

## 第四步：写 memory 字段

每条 `memory` 都必须包含：

- `type`
- `judgment`
- `reasoning`
- `tags`

### `memory.type` 可选值

- `knowledge`
- `skill`
- `event`
- `emotional`

默认优先使用 `knowledge`。

### `judgment`（结论）
- 必须是完整陈述句，脱离当前对话也能独立看懂。
- 一条记忆只表达一个论断，不要把两件事塞进一条。
- 记录结论、事实、稳定偏好、可复用方法，不记录过程流水账。
- 涉及具体群友时，必须准确写出真实昵称和真实用户 ID，格式：`小明（123456）`。
- 禁用「用户说过」「本轮对话」等过程词，不评价、不夸大、不无据推断。

### `reasoning`（依据）
- 只写事实来源：对话里哪些内容支撑了这个结论。
- 不写目的性文字（如「为了以后能用」），不写心理活动。
- 如果 `judgment` 已经完整、无需额外佐证，可写 `""`。
- 引用原话或专有名词时用 `「」`，不要用英文双引号。

### `tags`（检索锚点）
- 必须是简短可检索的节点词，不写短语或整句。
- 优先填明确实体：人物、群友 ID、项目、工具、技能、作品、地点、事件名。
- 除群友 ID 外，不要输出其它纯数字标签。
